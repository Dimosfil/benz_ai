# Контракт агрегации топлива

## Пользовательский результат

По запросу города, села или региона приложение возвращает найденные АЗС,
вероятностное наличие топлива, опубликованные цены, время свежести и состояние
каждого внешнего источника. Отсутствие цены не означает отсутствие топлива, а
наличие цены не подтверждает фактический остаток.

## Роли источников

- T-Bank Fuel: каталог АЗС и вероятностные статусы по видам топлива.
- Alfa AZS: общероссийский каталог, вероятностные статусы по видам топлива и
  опубликованные цены на основе транзакционной активности. Защитная HTTP-проверка
  завершается короткой серверной cookie-сессией без Chromium. Общий снимок
  кэшируется на 60 секунд, после чего станции фильтруются по bbox и
  административному контуру. Категория `AI98_100` сохраняется как единый вид
  `98/100`, потому что источник не разделяет эти марки.
- Sber AZS: независимый каталог и источник наличия. JSON endpoint станций
  подтверждён в настоящем Chromium-сеансе после JavaScript-проверки; прямой
  server-to-server запрос блокируется. Постоянное подключение требует
  согласованного доступа либо отдельного надёжного браузерного процесса.
- BenzUp: лицензируемый каталог АЗС и цены; требует серверный Bearer token.
- Multigo: серверный POST-каталог ближайших объектов категории АЗС. Запрос
  получает центр пользовательской области (`lat`, `lng`) и ограничение `limit`.
  Возвращаемый общий статус относится к объекту, а не к наличию топлива, поэтому
  источник дополняет каталог и не создаёт availability-сигнал. После ответа
  приложение оставляет только точки внутри административного контура найденной
  территории (или bbox, если контур недоступен) и исключает чистые ЭЗС без топлива.
  Запрос выполняется только по пользовательскому поиску
  и кэшируется на 60 секунд.
- Яндекс Карты: при каждом поиске проверяются карточки всех АЗС со статусом
  `available`, для которых известен `yandexOrgId`. Успешная проверка добавляет
  Яндекс к источникам конкретной АЗС и обновляет опубликованные цены, но не
  подтверждает наличие топлива и не меняет его вероятностный статус. Запросы
  можно отключить через `ENABLE_YANDEX_PRICES=0` или ограничить положительным
  `YANDEX_PRICE_LIMIT`.
- ГдеБЕНЗ: независимый краудсорсинговый сигнал наличия, очередей и лимитов.
  Проверенный read endpoint — `/api/nearby` с `lat`, `lon` и `radius_km`.
  Приложение делает запрос только вследствие пользовательского поиска и
  кэширует область на 60 секунд; фоновый обход регионов отсутствует. Радиусный
  ответ фильтруется по административному контуру найденной территории до агрегации.
- OpenStreetMap Nominatim: определение bbox и GeoJSON-контура территории, не
  источник топлива. Для Polygon и MultiPolygon учитываются внешние кольца и
  внутренние исключения; при отсутствии поддерживаемого контура остаётся bbox.

## Инварианты

- Секреты хранятся только в серверных переменных окружения.
- Ошибка одного источника не блокирует ответы других источников.
- Частичный отказ всегда подписывается именем источника; сырой browser/runtime
  текст вида `Failed to fetch` не показывается пользователю без контекста.
- При полном отказе API веб-интерфейс очищает прежнюю сводку, чтобы устаревший
  результат не выглядел актуальным.
- Sber Chromium-worker запускается в headless-режиме без GUI, использует один
  временный браузерный профиль, обновляет только активные области раз в 60
  секунд и удаляет область после 15 минут без запросов.
- Cookies Sber не записываются в проект, ответы браузера не исполняются вне
  изолированной страницы источника, временный профиль удаляется при остановке.
- Cookies Alfa хранятся только в памяти процесса и используются исключительно
  для завершения защитной HTTP-проверки; глобальный снимок не запрашивается для
  каждой территории повторно в пределах TTL.
- ГдеБЕНЗ не опрашивается фоновым краулером; один пользовательский поиск имеет
  один кэшируемый запрос к источнику.
- API и интерфейс явно показывают недоступные и неподключённые источники.
- Наличие всегда формулируется вероятностно и сохраняется отдельно от цены.
- Время операции/наблюдения наличия и время обновления цены не объединяются.
- Отсутствующий в ответе Sber счётчик операций остаётся неизвестным и не
  отображается как ноль; `0 операций` допустимо показывать только при явно
  переданном источником нулевом значении.
- Неизвестный ответ внешнего API считается ошибкой контракта, а не пустым
  успешным результатом.
- Фильтры топлива и статусов поддерживают мультивыбор: несколько отмеченных
  значений объединяются по правилу «ИЛИ», а `Любое` / `Все статусы` очищают
  остальные галочки. Выбор всех известных видов топлива также нормализуется к
  `Любое`, поэтому АЗС без разбивки по топливу не исчезают из выдачи. Без
  конкретных значений фильтр не ограничивает выдачу.
- Карта и текстовая сводка имеют разные области загрузки. Таблица и мобильные
  карточки остаются полным результатом выбранного города/региона, а карта после
  завершённого перемещения с задержкой работает через скользящий пространственный
  кэш. Первичный bbox расширяется на 45% размера экрана с каждой стороны. Пока
  экран остаётся внутри этой загруженной рамки, новый запрос не выполняется; при
  выходе за неё клиент запрашивает только новые верхнюю, нижнюю, левую или правую
  полосы и объединяет ответ с уже показанными АЗС. Маркеры не очищаются во время
  догрузки. Станции удаляются из клиентского кэша только после выхода за рамку,
  расширенную на один размер экрана с каждой стороны; фильтры и счётчик при этом
  относятся к фактически видимой области. Новый запрос браузера отменяет ещё не
  завершённый предыдущий. При масштабе меньше 8 загрузка не выполняется; API
  отклоняет bbox со стороной более 12 градусов. При первом
  открытии браузер запрашивает геопозицию и, если пользователь разрешил доступ,
  центрирует карту на ней; кнопка геолокации позволяет повторить действие.
  Явный текстовый поиск по территории снова переводит карту к найденным АЗС.
  Карта занимает всю высоту своей панели без отдельного заголовка над ней;
  счётчик видимых АЗС показывается компактным индикатором поверх карты.
  Близкие точки объединяются в кластеры. Кластер показывает количество АЗС в
  центре, а его кольцевая круговая диаграмма пропорционально отображает статусы
  дочерних АЗС: зелёный, оранжевый, красный и серый сегменты используют те же
  значения, что отдельные маркеры. Однородный кластер целиком окрашивается в
  соответствующий цвет. Цвет отдельного маркера вычисляется
  по тем же выбранным видам топлива и тем же вероятностным статусам, что таблица
  и мобильные карточки. Карточка маркера показывает агрегированные источники,
  адрес, статус, топливо, цены, свежесть, детали и доступные ссылки на внешние
  карты; процент согласованности показывается только при наличии отдельных
  availability-сигналов источников. Недоступность клиентской картографической библиотеки не блокирует
  список АЗС.
- `mode=viewport` сохраняет параллельный сбор всех основных провайдеров, но
  ограничивает ожидание каждого из них 12 секундами по умолчанию и пропускает
  последовательное обогащение Яндексом. Поэтому медленный источник становится
  подписанным частичным отказом, а не удерживает весь ответ карты. Полная сводка
  города/региона продолжает выполнять ценовое обогащение. Клиент отменяет
  viewport-запрос через 18 секунд и заменяет загрузочный overlay понятной
  ошибкой; новое перемещение карты запускает повторную попытку.
- Sber Chromium-worker ограничивает и browser fetch, и каждую CDP-команду.
  Тайм-аут browser fetch настраивается через `SBER_REQUEST_TIMEOUT_MS`; зависший
  DevTools-вызов не может бессрочно удерживать worker operation.
- Серверные кэши динамических bbox ограничены по числу записей и используют LRU
  при чтении. По умолчанию общая выдача хранит не более 32 областей, bbox-кэши
  ГдеБЕНЗ и Multigo — не более 64 областей каждый, а ценовые карточки Яндекса —
  не более 1000 станций. Лимиты настраиваются переменными окружения.
- Таблица АЗС сортируется по любой колонке, выводится страницами по
  10/25/50/100 строк (25 по умолчанию), имеет липкую шапку и отдельную
  вертикальную прокрутку. Колонки распределяются по доступной ширине без
  горизонтальной прокрутки. Поиск, фильтр и новая сортировка сбрасывают номер
  страницы; мобильные карточки используют ту же страницу и порядок строк.
- Пользователь может менять ширину и высоту окна таблицы за правый нижний угол,
  а также перетаскивать заголовки, меняя порядок столбцов.
  Размер и прокрутка таблицы, территория, поисковая строка, фильтры, сортировка,
  страница, размер страницы и порядок столбцов сохраняются локально и
  восстанавливаются после F5.
  Сами данные АЗС не дублируются в браузерном хранилище и загружаются из
  серверного кэша, чтобы не показывать устаревшую копию как актуальную.
- Результаты разделены на две доступные с клавиатуры вкладки: полноэкранную
  карту с её счётчиком и легендой и отдельную таблицу со сводными виджетами.
  Успешный явный поиск открывает карту и подбирает её границы по АЗС, которые
  соответствуют территории, текстовому запросу и выбранным фильтрам. Выбранная
  вкладка сохраняется локально и восстанавливается после F5.
  Найденная территория, краткий счётчик и переключатель карты/таблицы находятся
  в общей шапке и не исчезают при смене представления. Поиск и фильтры вынесены
  в правую колонку рабочей области. Документ занимает ровно высоту окна и не
  прокручивается целиком; карта растягивается на свободное место, а длинная
  таблица и правая панель прокручиваются только внутри своих областей.
- Зелёный статус «Вероятно есть» используется только при минимум двух
  согласованных положительных сигналах и согласованности не ниже 80%.
  Одиночный положительный сигнал и согласованность 50% показываются жёлтым
  статусом «Возможно есть». В карточке АЗС отдельно показывается самая свежая
  известная оплата из банковских источников; время пользовательского отчёта
  ГдеБЕНЗ не называется оплатой.
- Пользовательская команда «Обновить весь кэш» очищает сводку территории,
  геокодирование, Alfa, ГдеБЕНЗ, Sber и ценовой кэш Яндекса, затем синхронно собирает
  текущий запрос заново. Chromium cookies не очищаются: это временная сессия
  доступа, а не кэш данных.
- Экспериментальный источник ограничивает количество и параллелизм запросов и
  кэширует ответы.

## Проверка

- Запрос по Бабяково должен возвращать станции T-Bank даже без дополнительных
  ключей.
- Без `BENZUP_API_TOKEN` BenzUp имеет состояние `configured: false`.
- При `ENABLE_YANDEX_PRICES=0` Яндекс имеет состояние `configured: false`; по
  умолчанию проверка включена.
- При разрешённом включении Яндекс цены возвращаются отдельно в `prices`.
- Sber должен возвращаться как доступный только после успешного JSON-ответа из
  Chromium-worker; ошибка browser-worker не должна блокировать другие источники.
- Alfa должен завершать cookie-проверку, валидировать массив станций, сохранять
  цены отдельно от availability и отбрасывать точки вне запрошенного bbox.
- ГдеБЕНЗ должен сохранять `detail`, `confirmations`, `confidence` и исходный
  статус отдельно от сводного статуса.
- Записи с разными внешними ID одного провайдера не объединяются только из-за
  близких координат или совпавшего названия. Это исключает дубли источника вида
  `Multigo + Multigo`; записи разных провайдеров по-прежнему могут объединяться.
- Исключение составляют доказанные дубли каталога: записи с разными ID одного
  провайдера объединяются, когда их координаты находятся не далее 5 метров и
  нормализованный адрес полностью совпадает. Все внешние ID сохраняются, а
  evidence одного источника объединяется без потери видов топлива и самой
  свежей временной метки. Свежий положительный сигнал независимого источника
  применяется к общей физической АЗС во всех API и клиентских интерфейсах.
  Названия и адреса всех объединённых карточек сохраняются как алиасы и
  участвуют в текстовом поиске; для отображения выбирается наиболее конкретное
  название бренда без служебного типа объекта или номера точки.
- Итоговая сводка территории не включает точки любых источников за пределами
  административного контура, даже если bbox/nearest/radius endpoint вернул их
  в исходном ответе. Bbox остаётся fallback только при отсутствии GeoJSON.
