# Telegram-интерфейс Benz AI

## Роль

Telegram и веб — два клиентских адаптера одной серверной логики поиска топлива.
Telegram не содержит собственных правил агрегации и вызывает тот же use case
`summaryFor`, что и `GET /api/summary`. Веб остаётся полным интерфейсом с
таблицей и фильтрами; Telegram возвращает короткую текстовую сводку.

## Пользовательский поток

1. `/start` или `/help` объясняет, что нужно отправить город либо регион России.
2. Обычный текст передаётся в общий поиск территории и АЗС.
3. Бот показывает число найденных АЗС, вероятностные статусы, наличие цен,
   краткую разбивку по видам топлива и до восьми наиболее полезных карточек
   АЗС с адресом, топливом, ценами, свежестью и источниками.
4. Ограничения источников и вероятностная природа данных остаются видимыми.
5. Неизвестная slash-команда не запускает поиск и возвращает подсказку.

## Runtime-контракт

- `TELEGRAM_POLLING_ENABLED=false` по умолчанию.
- `TELEGRAM_BOT_TOKEN` обязателен только при включённом polling.
- `.env` загружается Node.js через `--env-file-if-exists=.env` в npm scripts.
- Gateway использует Telegram Bot API `getUpdates` и `sendMessage`.
- Offset увеличивается до `update_id + 1`, чтобы обработанные обновления не
  запрашивались повторно.
- Ошибка polling записывается в безопасный `/api/health` и приводит к повтору
  после задержки; токен в статус и ответы не попадает.
- Сетевой вызов polling ограничен временем Telegram long poll плюс запас на
  подключение; отправка ответа также имеет timeout. Зависшее соединение не
  должно бесконечно оставлять ложный зелёный статус.
- `/api/health` показывает время последнего завершённого poll, последнего
  обработанного update и счётчик обработанных updates без пользовательских
  данных.
- При остановке HTTP-сервера активный long-poll запрос отменяется.
- Один Telegram bot token обслуживается одной активной polling-репликой.
  Горизонтальное масштабирование требует перехода на webhook или отдельного
  singleton gateway-worker.
- DeepSeek не вызывается текущим Telegram-поиском. Его использование требует
  отдельного AI-use-case и не должно подменять фактические данные источников.

## Источник

Логика адаптирована из `D:\AI\telegram_bot_template`, прежде всего из
`apps/api/src/telegramPollingGateway.ts`, config/DTO-контрактов и focused tests.
Шаблонная бизнес-логика не переносилась.

## Проверка

- токен проверяется до запуска polling;
- текст Telegram нормализуется в клиент-независимый вход;
- город передаётся в общий summary use case;
- ответ отправляется в исходный `chat_id`;
- формулировки не превращают вероятностные данные в гарантию наличия.
