# Контракт интеграции DeepSeek

## Назначение

В Benz AI принят только серверный транспорт DeepSeek из внешнего проекта
`D:\AI\llm_providers`. Остальные провайдеры, реестр выбора провайдера, Codex,
mock-реализация и общий пакет не переносятся.

Адаптер является подготовленной интеграционной границей. Он не участвует в
поиске АЗС и не вызывается пользовательским интерфейсом, пока отдельная
продуктовая функция явно не определит промпт, входные данные, бюджет и способ
показа результата.

## Контракт

- Точка входа: `createDeepSeekProvider()` в `providers/deepseek.js`.
- Транспорт: `POST <DEEPSEEK_BASE_URL>/chat/completions`.
- Аутентификация: серверный Bearer-токен из `DEEPSEEK_API_KEY`.
- Модель по умолчанию: `deepseek-chat`; переопределяется через
  `DEEPSEEK_MODEL` или в конкретном запросе.
- Вход: `prompt` либо непустой массив `messages`; дополнительно поддерживаются
  `temperature`, `maxTokens` и `json`.
- Выход: `{ provider, model, output, raw }`; пустой `choices[0].message.content`
  считается ошибкой контракта.
- HTTP-ошибка и таймаут завершают только вызов DeepSeek. Адаптер не изменяет
  данные АЗС и не подменяет фактические сигналы наличия топлива выводом модели.

## Инварианты безопасности

- API-ключ не передаётся в браузер, ответы API, логи или project memory.
- Конфигурация хранит только имя переменной окружения и безопасные значения по
  умолчанию.
- Вызовы не запускаются в фоне и не добавлены в основной поиск АЗС.

## Источник и доказательства

- Внешний источник: `D:\AI\llm_providers`.
- Изучены: `README.md`, `package.json`, `src/config.mjs`,
  `src/contracts.mjs`, `src/providerRegistry.mjs`,
  `src/providers/deepseek/index.mjs`,
  `src/providers/openai-compatible/index.mjs` и соответствующие provider tests.
- Перенесённый контракт: OpenAI-совместимое тело запроса, Bearer-аутентификация,
  значения DeepSeek по умолчанию, проверка HTTP-ответа и извлечение текстового
  результата.
- Локальная адаптация: стандартный таймаут Benz AI и отсутствие общей
  мультипровайдерной абстракции.

## Проверка

`npm test` должен подтверждать сборку тела запроса, серверную авторизацию,
извлечение ответа и состояние `isConfigured()` без ключа.
